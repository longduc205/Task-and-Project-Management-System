## Scheduling Tools: The Gantt Chart

Named after Henry L. Gantt (who developed it for the WWI munitions program), this is the most common visual scheduling technique. Time is the common denominator. Start and completion times of work packages are indicated by the beginning and ending of each bar. Used to easily assess progress vs. time.

Project Planning is not a collection of isolated documents. It is a unified system.

## Beyond the Gantt Chart

- Gantt charts show time.
- Networks explicitly show dependencies.
- Networks reveal delay impacts.
- Vital for project control.

## Activity-on-Node Networks

- Nodes represent work tasks.
- Arrows show logical sequence.
- Predecessors must finish first.
- Successors follow completed tasks.

![Image](/Users/azure_awabi/Downloads/Task-and-Project-Management-System/markdown_docs/04 Project Planning and Scheduling_artifacts/image_000039_c7559df1f718968d012483c5dd82dd232d5198b4b99347d736d89ff52c899a07.png)

( ) = Duration in seconds

## Building the Project Network

- Start with one node.
- Connect immediate predecessors only.
- Avoid redundant dependency lines.
- End with one node.

![Image](/Users/azure_awabi/Downloads/Task-and-Project-Management-System/markdown_docs/04 Project Planning and Scheduling_artifacts/image_000040_85580b6afe6dfa465f1deb8de2a9840d7b40ecd803cd77aaa58b059a1d312c42.png)

## Defining the Critical Path

- Longest path through network.
- Determines shortest project duration.
- Dictates final target deadlines.
- Zero schedule flexibility allowed.

![Image](/Users/azure_awabi/Downloads/Task-and-Project-Management-System/markdown_docs/04 Project Planning and Scheduling_artifacts/image_000041_b7362e97e138adabd2df13d1582299bb282b1b799141d35de0e07c7721d8bc3a.png)

## The Forward Pass

- Moves left to right.
- Calculates early start times.
- Calculates early finish times.
- ES plus duration equals EF.

![Image](/Users/azure_awabi/Downloads/Task-and-Project-Management-System/markdown_docs/04 Project Planning and Scheduling_artifacts/image_000042_74ffa49eae6e0f245183532e74a73638e2438c7d8167587b2aefe15ac5b51fd4.png)

EF = Early Finish

## The Backward Pass

- Moves right to left.
- Calculates late finish times.
- Calculates late start times.
- LF minus duration equals LS.

Marigold Arrow Indicates Backward Pass

![Image](/Users/azure_awabi/Downloads/Task-and-Project-Management-System/markdown_docs/04 Project Planning and Scheduling_artifacts/image_000043_a4e8947da640c026ac3116b646bb35c4727e08f2c886a121e6270000677124e5.png)

## Calculating Slack and Float

- Measures schedule flexibility.
- Total Slack: Overall flexibility.
- Delays without delaying project.
- Free Slack: Task flexibility.
- Delays without delaying successors.
- Critical path has none.

![Image](/Users/azure_awabi/Downloads/Task-and-Project-Management-System/markdown_docs/04 Project Planning and Scheduling_artifacts/image_000044_04f8a63c8398a48a503d12a04dfbb5fb8bc77011f5e141935782f10049912698.png)
